# Arquitecto AI

## Visión

Arquitecto AI es una plataforma que utiliza Inteligencia Artificial para asistir en el diseño de sistemas de software.

Su propósito es transformar una idea de negocio en una arquitectura técnica bien definida, permitiendo reducir el tiempo de análisis y facilitar la toma de decisiones durante las primeras etapas de un proyecto.

El sistema debe comportarse como un Arquitecto de Software Senior, guiando al usuario mediante preguntas, validando requerimientos y generando documentación técnica antes de producir cualquier código.

---

# Objetivos

- Guiar al usuario durante el análisis de requerimientos.
- Detectar información faltante.
- Diseñar arquitecturas de software.
- Proponer entidades y relaciones.
- Generar documentación técnica.
- Facilitar la generación posterior de código.

---

# Alcance inicial

La primera versión del sistema estará enfocada únicamente en el proceso de análisis y diseño.

No se busca reemplazar al desarrollador, sino asistirlo durante la etapa de arquitectura.

---

# Roadmap

## Fase 0

Preparación del proyecto.

Objetivos:

- Crear la estructura del repositorio.
- Configurar el entorno de desarrollo.
- Definir la arquitectura del proyecto.
- Configurar las herramientas necesarias.

Estado:
- Completada

---

## Fase 1

Análisis de requerimientos.

El usuario proporcionará:

- Objetivo del sistema.
- Descripción general.
- Requerimientos.
- Tecnologías sugeridas (opcionales).

El sistema deberá:

- Identificar requerimientos funcionales.
- Identificar requerimientos no funcionales.
- Detectar información faltante.
- Formular preguntas.
- Construir un documento de análisis.

Salida esperada:

- Documento de requerimientos.
- Lista de dudas.
- Resumen del sistema.

---

## Fase 2

Diseño funcional.

El sistema deberá generar:

- Módulos.
- Casos de uso.
- Entidades.
- Relaciones.
- Reglas de negocio.

Salida esperada:

- Documento funcional.
- Modelo conceptual.

---

## Fase 3

Diseño técnico.

El sistema deberá generar:

- Arquitectura.
- APIs.
- Base de datos.
- Componentes.
- Diagramas.

Salida esperada:

- Documento técnico.
- Diagrama de arquitectura.

---

## Fase 4

Generación de código.

El sistema podrá generar:

- Backend.
- Frontend.
- Base de datos.
- Pruebas.

---

## Fase 5

Sistema multiagente.

Especialización de agentes:

- Arquitecto.
- Analista.
- Revisor.
- Diseñador de Base de Datos.
- Generador de Código.

---

# Principios

Todo diseño debe ser:

- Escalable.
- Mantenible.
- Comprensible.
- Justificado.

Cada decisión técnica deberá incluir una explicación de por qué fue tomada.
